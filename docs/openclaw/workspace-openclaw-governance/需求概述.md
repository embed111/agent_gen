# 需求概述

## 1. 项目背景与问题定义
当前 `agent_gen` 是配置驱动的 Agent 生成工作区，具备完整生成与校验流程，但尚未具备 OpenClaw 风格 `.codex/` 记忆层。

## 2. 目标用户与核心场景
1. 目标用户：生成 agent、owner、执行维护者。
2. 核心场景：
   - 保留配置优先的生成模式；
   - 新增身份、用户、长期/每日记忆能力；
   - 将生成经验沉淀为 agent 个体记忆。

## 3. 业务目标与成功指标
1. 业务目标：为 `agent_gen` 补齐 `.codex/` 记忆层。
2. 成功指标：
   - `.codex/` 核心文档与 `memory/` 齐全；
   - `AGENTS.md` 已补齐读取顺序；
   - 现有生成流程不回退。

## 4. 范围界定
### In Scope
1. 新增 `.codex/`。
2. 更新 `AGENTS.md` 读取顺序。

### Out of Scope
1. 配置模型、生成脚本、测试框架重构。

## 5. 优先级与阶段规划
1. P0：补 `.codex/`。
2. P1：更新 `AGENTS.md`。
3. P2：做最小生成回归。

## 6. 风险、依赖与待确认事项
1. 风险：把正式报告/测试产物写进记忆层。
2. 依赖：执行方具备更新 `AGENTS.md` 与新增目录权限。
